Keeping track of movies with Emacs and Org mode

Feb 3 2021

Introduction#

A while back, I decided to start cataloging the movies I’ve seen. This started as a simple table in Emacs Org mode . Below is an example of the table’s structure:


Title Year Genre imdbRating Rating Notes
Up 2009 Animation, Adventure, Comedy 8.2 8 / 10 Lots of Balloons
WALL-E 2008 Animation, Adventure, Family 8.4 8.3 / 10 Littering is bad
Toy Story 1995 Animation, Adventure, Comedy 8.3 8.5 / 10

Manually entering metadata for each movie was tedious, especially since I wanted to backfill movies I’d previously seen. I wanted to simply add the title and have the rest of the information filled in for me. I googled around for a solution and found this Gist which demonstrates how to import movie metadata from OMDB (“The Open Movie Database”) into Org mode (specifically, a nicely formatted IMDB link with some metadata attached to it). Although not exactly what I want to do, it did introduce me to what OMDB was and how one can query it with a little bit of Elisp.

From there, I wrote some Org Babel code to return specific metadata properties (e.g., “Year”, “Genre”, “imdbRating”) for a given movie. Finally, using Org mode’s spreadsheet capabilities , I was able to automatically populate each row with the corresponding movie’s metadata, as well as create a nicely formatted IMDB link in the “Title” column. A demo can be seen below and the rest of the article goes over the OMDB API and Org file setup.


Using OMDB’s API#

Using OMDB’s API is pretty simple. First, sign-up here to receive an API key. The free-tier offers 1000 requests per day, which should be plenty for this project.

Once you have a key, you can start making requests. For example, running:

GET http://omdbapi.com/?t=Up&apiKey=[YOUR KEY]

Yields:

{
   "Title":"Up",
   "Year":"2009",
   "Rated":"PG",
   "Released":"29 May 2009",
   "Runtime":"96 min",
   "Genre":"Animation, Adventure, Comedy, Family",
   "Director":"Pete Docter, Bob Peterson(co-director)",
   "Writer":"Pete Docter (story by), Bob Peterson (story by), Tom McCarthy (story by), Bob Peterson (screenplay by), Pete Docter (screenplay by)",
   <snip>
}

You can also optionally specify the year with the y query parameter. This will help disambiguate movies with the same title that were released in different years. If two movies have the same name and were both released in the same year1 - well, lets now worry about that for now 😄.

The code#

Below is a Gist with a sample Org file to get started. To see the raw Org file, you can click “view raw” in the bottom right. A few notes:

  1. To use your OMDB key, you can either put it in the file ~/.omdbapikey or directly set the omdb-api-key variable.

  2. To avoid constant prompts to evaluate inline code within the Org file, you may wish to set org-confirm-babel-evaluate to nil. Of course, make sure you read and trust my code and review this page first.

  3. In order for the IMDB links to work, make sure you evaluate (C-c C-c) the line #+LINK: imdb https://www.imdb.com/title/%s.

  4. The entry point for the table formula to retrieve a cell’s details is get-movie-metadata. This function takes the movie’s title, the movie’s release year (optional, if disambiguation is needed), and the header name (e.g., Genre, imdbRating). It returns the metadata that corresponds to the column’s header.

  5. The code maintains a simple cache represented as a 2-level hash table (first level is the movie title and the second level is the release year). This is to avoid calling OMDB multiple times for the same movie, since we only need 1 request to retrieve all of the metadata for the row.

  6. The table formula at the bottom is a bit cryptic, so I’ll briefly go through the pieces:

    #+TBLFM: @2$<..@>$4='(if (or (string-blank-p $0) (and (equal @1 "Title") (not (string-prefix-p "[[" $0)))) (org-sbe "get-movie-metadata" (title $$1) (year $$2) (header $@1)) $0)

    a. @2$<..@>$4=: This will restrict the range of the formula to the second row onwards (ignoring the header) and from columns 1 to 4. The 5th and 6th columns are where I write my own notes. If you wish to add more metadata from the movie, you can simply add another column with the appropriate header (e.g., “Director”) and modify the range to include the new column.

    b. (or (string-blank-p $0) (and (equal @1 "Title") (not (string-prefix-p "[[" $0)))): This conditional is an optimization to only run the Org babel block if the cell is either empty or if it is the “Title” column and it hasn’t been formatted as an IMDB link yet, which will start with “[[”. If this condition is false, the cell is populated with whatever was already there, which is represented by the special $0 cell reference.

    c. (org-sbe "get-movie-metadata" (title $$1) (year $$2) (header $@1)): This calls the Org babel function and populates the cell with the result.
